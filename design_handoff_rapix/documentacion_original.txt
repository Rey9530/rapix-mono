DOCUMENTACION DE PANTALLAS - APP_CLIENTES (RAPIX)
==================================================

Audiencia: insumo para un modelo de diseno UI/UX (Claude Design).
Strings de UI reportados tal cual aparecen en el codigo (sin tildes en
muchos casos, segun convencion del proyecto). Las observaciones inferidas
se marcan como "(supuesto)".


INDICE DE PANTALLAS

  1. Iniciar sesion              /iniciar-sesion           publica
  2. Registrar (vendedor)        /registrar                publica
  3. Selector de ubicacion       /seleccionar-ubicacion    publica (modal)
  4. Inicio (dashboard)          /inicio                   autenticada
  5. Listado de pedidos          /pedidos                  autenticada
  6. Crear pedido                /pedidos/nuevo            autenticada
  7. Detalle de pedido           /pedidos/:id              autenticada
  8. Mis paquetes                /paquetes                 autenticada
  9. Tienda de paquetes          /paquetes/tienda          autenticada
 10. Mi perfil                   /perfil                   autenticada
 11. Editar negocio              /perfil/editar-negocio    autenticada
 12. Seguimiento publico         /seguimiento/:codigo      publica (deep link)


CONVENCIONES GENERALES

  Stack: Flutter + Dart, Material 3, Riverpod, go_router (StatefulShellRoute),
  Mapbox, Dio, image_picker.

  Guard de auth (lib/nucleo/enrutador/enrutador_app.dart):
    - Publicas: /iniciar-sesion, /registrar, /seleccionar-ubicacion,
      /seguimiento/:codigo
    - No autenticado en ruta privada -> redirige a /iniciar-sesion
    - Autenticado en /iniciar-sesion o /registrar -> redirige a /inicio
    - initialLocation: /inicio
    - 4 ramas (tabs): /inicio, /pedidos, /paquetes, /perfil

  Patrones globales:
    - Pull-to-refresh en listados y detalles
    - SnackBar para errores (mensaje extraido de DioException o fallback)
    - CircularProgressIndicator centrado al cargar; spinner 20x20 en botones
    - Form + GlobalKey<FormState> con validators por campo
    - Bloqueo de doble envio con bandera local _enviando o estado.cargando

1. INICIAR SESION
==================================================================

  Clase: IniciarSesionPantalla (ConsumerStatefulWidget)
  Ruta:  /iniciar-sesion
  Archivo: lib/caracteristicas/autenticacion/iniciar_sesion_pantalla.dart

  Proposito: pantalla de entrada principal. Login con email + contrasena
  o acceso al flujo de registro.

  Layout: icono local_shipping_outlined (64px), titulo "Bienvenido",
  subtitulo "Inicia sesion para gestionar tus pedidos", formulario centrado
  (max 420px), padding 24, scroll para pantallas pequenas.

  Campos:
    - Email (requerido):
        label "Correo electronico", icon email_outlined, autofill email
        Vacio        -> "Ingresa tu correo"
        Sin "@"      -> "Correo invalido"
        Trim antes de enviar
    - Contrasena (requerida):
        label "Contrasena", icon lock_outline, suffixIcon dinamico
        (visibility/visibility_off, toggle local _verContrasena)
        Vacia        -> "Ingresa tu contrasena"

  Acciones:
    - Toggle ojo: alterna obscureText
    - Boton "Iniciar sesion" (FilledButton): valida + llama
      autenticacionControlador.iniciarSesion(email.trim(), contrasena).
      Mientras carga: spinner; deshabilitado.
      Exito -> go('/inicio')
    - Boton "Crear cuenta de vendedor" (TextButton): push('/registrar')

  Navegacion:
    Llega: ruta inicial sin sesion; tras cerrar sesion en /perfil.
    Sale a: /inicio (login OK), /registrar.

  Estados:
    - Inicial / Cargando / Error validacion / Error servidor / Exito
    - Errores servidor (controlador): 401 -> "Credenciales invalidas";
      generico -> "No fue posible completar la operacion. Intenta de nuevo."

  Notas UX:
    (supuesto) Falta enlace "Olvidaste tu contrasena?".
    (supuesto) Validacion de email demasiado laxa: solo exige "@".
    SafeArea + ConstrainedBox(maxWidth: 420) para tablets.

2. REGISTRAR (CREAR CUENTA DE VENDEDOR)
==================================================================

  Clase: RegistrarPantalla (ConsumerStatefulWidget)
  Ruta:  /registrar
  Archivo: lib/caracteristicas/autenticacion/registrar_pantalla.dart

  Proposito: alta de nuevo vendedor. Datos personales, credenciales,
  datos del negocio y ubicacion geografica del local.

  Layout: AppBar "Crear cuenta de vendedor", scroll vertical, dos
  secciones separadas por Divider(32) con encabezado "Datos de la tienda".

  Campos:
    - Nombre completo (requerido, TextCapitalization.words):
        label "Nombre completo", icon person_outline
        length<2 -> "Requerido"
    - Correo electronico (requerido):
        label "Correo electronico", icon email_outlined
        Sin "@" -> "Correo invalido"
    - Telefono (requerido):
        label "Telefono", hint "+50370001234", icon phone_outlined
        Regex ^\+?[0-9]{8,15}$ -> "8-15 digitos, opcional con +"
    - Contrasena (requerida):
        label "Contrasena", icon lock_outline,
        helper "8+ chars, mayuscula, numero y simbolo"
        length<8                  -> "Minimo 8 caracteres"
        sin [A-Z]                 -> "Debe incluir una mayuscula"
        sin [0-9]                 -> "Debe incluir un numero"
        sin [^A-Za-z0-9]          -> "Debe incluir un simbolo"
    - Nombre del negocio (requerido, words):
        label "Nombre del negocio", icon storefront_outlined
        Vacio -> "Requerido"
    - Direccion (requerida):
        label "Direccion", icon place_outlined
        Vacio -> "Requerido"
    - Ubicacion (requerida, Card+ListTile):
        Sin: "Seleccionar ubicacion en mapa"
        Con: "Ubicacion seleccionada" + "(lat:.5, lng:.5)"
        Trailing chevron_right
        Validacion manual al enviar -> SnackBar
        "Selecciona la ubicacion de la tienda"

  Acciones:
    - Tap card ubicacion: push<mb.Point>('/seleccionar-ubicacion',
      extra: {titulo: "Ubicacion de la tienda --", inicial: <Point?>})
    - Boton "Crear cuenta" (FilledButton): valida -> verifica ubicacion ->
      autenticacionControlador.registrar(...) con lat/lng.
      Spinner durante envio. Exito -> go('/inicio').
    - Volver (AppBar): pop a /iniciar-sesion.

  Estados: Inicial / Cargando / Error ubicacion (SnackBar) /
    Error validacion / Error servidor (401, 409, PEDIDO_ZONA_INVALIDA,
    generico) / Exito.

  Notas UX:
    (supuesto) Sin medidor visual de fortaleza de contrasena.
    (supuesto) Sin confirmar contrasena.
    (supuesto) Titulo del selector con artefacto " --" al final.
    Coordenadas mostradas con 5 decimales.

3. SELECTOR DE UBICACION
==================================================================

  Clase: SelectorUbicacionPantalla (StatefulWidget)
  Ruta:  /seleccionar-ubicacion
  Archivo: lib/caracteristicas/autenticacion/selector_ubicacion_pantalla.dart

  Proposito: modal reutilizable para elegir ubicacion en mapa Mapbox
  (pin fijo en el centro mientras el usuario mueve el mapa). Fallback
  manual si Mapbox no esta configurado. Retorna mb.Point via context.pop().

  Parametros:
    - titulo (default "Seleccionar ubicacion")
    - inicial (mb.Point?) - si es null usa Entorno.latitudInicial / longitudInicial

  Layout modo mapa (token disponible):
    - AppBar con widget.titulo
    - MapWidget pantalla completa (zoom inicial 15)
    - Pin central fijo: location_pin rojo 56px (IgnorePointer, padding inf 32)
    - Bottom card con coords "(lat:.5, lng:.5)" y boton

  Layout modo fallback (sin token):
    - Card aviso: "Mapbox no esta configurado. Ingresa las coordenadas
      manualmente o establece MAPBOX_TOKEN al compilar."
    - 2 TextField numericos signed/decimal: "Latitud" y "Longitud"
    - Boton "Confirmar ubicacion"

  Acciones:
    - Mover mapa: onCameraChangeListener actualiza _centroActual
    - Boton "Confirmar ubicacion" (mapa) -> context.pop(_centroActual)
    - Boton "Confirmar ubicacion" (fallback): tryParse lat/lng;
      si validos -> context.pop(Point(Position(lng, lat)))

  Navegacion:
    Llega: desde /registrar y /perfil/editar-negocio.
    Sale: pop con mb.Point.

  Notas UX:
    Patron pin fijo + mapa movil convencional y funcional.
    (supuesto) Sin buscador por direccion (geocoding).
    (supuesto) Sin "usar mi ubicacion actual" (GPS).
    (supuesto) Controllers locales del fallback no se disponen (leak menor).

6. CREAR PEDIDO
==================================================================

  Clase: CrearPedidoPantalla (ConsumerStatefulWidget)
  Ruta:  /pedidos/nuevo
  Archivo: lib/caracteristicas/pedidos/crear_pedido_pantalla.dart

  Proposito: formulario para registrar nuevo pedido. Datos del cliente,
  destino con URL Google Maps validada, foto opcional, metodo de pago
  y monto si aplica. La ubicacion de origen se toma del perfil vendedor.

  Layout: AppBar "Nuevo pedido". Form con tres secciones separadas
  por Divider(32): "Datos del cliente", "Destino", "Paquete".

  Campos:
    Datos del cliente:
      - Nombre completo (req, words):
          label "Nombre completo", icon person
          Trim vacio -> "Requerido"
      - Telefono (req):
          label "Telefono", hint "+50370001234", icon phone
          Regex ^\+?[0-9]{8,15}$ -> "8-15 digitos, opcional con +"

    Destino:
      - Direccion (req):
          label "Direccion", icon place
          Trim vacio -> "Requerido"
      - URL de Google Maps (req):
          label "URL de Google Maps del destino"
          hint "https://maps.app.goo.gl/..."
          helper "En Google Maps: Compartir -> Copiar enlace"
          icon map
          Vacio -> "Pega la URL de Google Maps del destino"
          Regex https://maps\.app\.goo\.gl/[A-Za-z0-9_-]+/?$
              -> "URL invalida. Comparte el sitio desde Google Maps
                  y pega el enlace corto"
      - Notas (opcional):
          label "Notas para el repartidor (opcional)", icon note

    Paquete:
      - Descripcion (opcional):
          label "Descripcion (opcional)", icon inventory
      - Foto (opcional):
          Card "Foto del paquete (opcional)" /
          "Tomar foto o elegir de galeria"
          maxWidth 1600, calidad 80
      - Metodo de pago (req, default CONTRA_ENTREGA):
          SegmentedButton: "Contra entrega" (icon payments) /
          "Prepagado" (icon credit_card)
      - Monto a cobrar (condicional, solo si CONTRA_ENTREGA):
          label "Monto a cobrar", numerico decimal
          tryParse <=0 -> "Monto invalido"

  Acciones:
    - Tap card foto -> ModalBottomSheet:
        "Tomar foto" (icon camera) -> ImagePicker.camera
        "Elegir de galeria" (icon photo_library) -> ImagePicker.gallery
    - "Quitar" / "Cambiar" foto
    - Boton "Crear pedido" (FilledButton):
        valida form ->
        verifica perfilVendedor.lat/lng != null
          (si null: SnackBar "Tu cuenta no tiene ubicacion de tienda.
                              Actualiza tu perfil.") ->
        POST /pedidos (multipart) ->
        invalida pedidosListadoProvider ->
        go('/pedidos/{id}')

  Mapeo de errores backend (_mensajeError):
    PEDIDO_ZONA_INVALIDA / PEDIDO_ZONA_INVALIDA_DESTINO ->
        "La direccion de destino esta fuera de la zona de cobertura"
    PEDIDO_URL_MAPAS_INVALIDA ->
        "No se pudo leer la ubicacion desde la URL de Google Maps.
         Verifica el enlace."
    Otros -> "No se pudo crear el pedido. Intenta de nuevo."
    Error image picker -> "No se pudo abrir el selector: {error}"

  Notas UX:
    (supuesto) Friccion ALTA del campo URL Google Maps: obliga a
    salir de la app. El regex es estricto (solo maps.app.goo.gl).
    Considerar selector embebido / geocoding por direccion / tap-and-hold
    en mapa.
    (supuesto) Sin recorte de foto.
    (supuesto) Boton siempre habilitado: deshabilitar hasta requeridos
    completos.
    Selector de foto duplicado con el de Editar negocio.

7. DETALLE DE PEDIDO
==================================================================

  Clase: PedidoDetallePantalla (ConsumerWidget)
  Ruta:  /pedidos/:id  (param pedidoId String)
  Archivo: lib/caracteristicas/pedidos/pedido_detalle_pantalla.dart

  Proposito: vista completa de un pedido con seguimiento en vivo y
  comprobante de entrega cuando aplica. Polling cada 10s en estados
  activos.

  Layout: AppBar "Detalle del pedido" + accion "Refrescar" (icon refresh).
  RefreshIndicator + ListView (padding 16). Bloques:
    1. Encabezado: codigo seguimiento (headlineSmall) + EstadoChip
    2. Card Cliente: nombre, telefono
    3. Card Destino: direccion
    4. Card Pago: "Metodo: ...", "Cobro contra entrega: $..." (si aplica),
       "Tarifa: $..." (si disponible)
    5. Seguimiento en vivo (condicional)
    6. Comprobante de entrega (condicional)
    7. Linea de tiempo

  Secciones condicionales:
    Seguimiento en vivo (estado in {ASIGNADO, RECOGIDO, EN_TRANSITO,
                                    EN_PUNTO_INTERCAMBIO, EN_REPARTO}):
      MapaSeguimientoVivo (220px alto, radius 16). Polling 10s.
      Marcadores "Destino" + "Repartidor" (si tiene posicion).
      Camara animada (zoom 14, 800ms).
      Fallback sin token: Card "Mapa no disponible (configura
      MAPBOX_TOKEN para verlo)."
      Tarjeta repartidor (si repartidorEntrega != null):
        avatar delivery_dining, nombreCompleto, telefono.

    Comprobante (estado == 'ENTREGADO' y comprobantes no vacio):
      Image.network (220px, radius 12).
      Fallback de carga: container gris + image_not_supported_outlined.

  Linea de tiempo (LineaTiempoEstado):
    Vacia -> "Aun no hay eventos registrados."
    Con eventos -> hitos verticales con (estadoNuevo ?? tipo)
                   replaceAll('_', ' ') + fecha dd/MM/yyyy HH:mm + notas.

  Acciones:
    - Boton refresh (AppBar, tooltip "Refrescar"): invalida
      pedidoDetalleProvider(pedidoId)
    - Pull-to-refresh: idem

  Estados: Loading / Error (icon + mensaje) / Datos.

  Notas UX:
    (supuesto) Polling 10s podria reemplazarse con push/WebSocket.
    (supuesto) Timeline larga sin opcion de colapsar.
    (supuesto) Sin acciones del cliente (cancelar, reportar incidente).
    Formato moneda: NumberFormat.currency(symbol: '$').

        label "Dirección", icon location_on
        Trim<3  -> "Ingresa una dirección"

                                     |

                                     +-- exito --> /pedidos/{id}

                ENTREGADO -> seccion comprobante

                 +-- error -> SnackBar
